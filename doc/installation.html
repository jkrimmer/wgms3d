Installation (CMake)

wgms3d now uses CMake as the only supported build and installation system.

The former Autotools flow (configure / make / make install) is deprecated.

Quick Start

cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build -j
cmake --install build

To run the executable from the build tree:

./build/wgms3d

Custom Install Prefix

cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DCMAKE_INSTALL_PREFIX=$HOME/.local
cmake --build build -j
cmake --install build

Dependencies via CMake

wgms3d can fetch/build major dependencies into the build tree.

Main CMake options:

  • WGMS3D_WITH_SUPERLU_ARPACK=ON|OFF
  • WGMS3D_WITH_PETSC_SLEPC=ON|OFF
  • WGMS3D_WITH_MPI=ON|OFF
  • WGMS3D_VENDOR_BOOST=ON|OFF
  • WGMS3D_VENDOR_SUPERLU=ON|OFF
  • WGMS3D_VENDOR_ARPACK=ON|OFF
  • WGMS3D_VENDOR_BLAS_LAPACK=ON|OFF

Example (vendored Boost, SuperLU, and ARPACK):

cmake -S . -B build \
  -DCMAKE_BUILD_TYPE=Release \
  -DWGMS3D_WITH_SUPERLU_ARPACK=ON \
  -DWGMS3D_VENDOR_BOOST=ON \
  -DWGMS3D_VENDOR_SUPERLU=ON \
  -DWGMS3D_VENDOR_ARPACK=ON
cmake --build build -j

PETSc / SLEPc Backend

For PETSc/SLEPc builds, disable SuperLU/ARPACK and enable MPI:

PETSC_DIR=/path/to/petsc SLEPC_DIR=/path/to/slepc \
cmake -S . -B build \
  -DWGMS3D_WITH_MPI=ON \
  -DWGMS3D_WITH_PETSC_SLEPC=ON \
  -DWGMS3D_WITH_SUPERLU_ARPACK=OFF
cmake --build build -j

Convenience Script

You can also run ./installScript.sh which wraps CMake configure/build/install.
